# Feature Spec: VOX DSL 文本插值支持完整表达式

**Feature**: `001-dsl-expr-interpolation`  
**Summary**: 在剧本文本中，将 `${变量}` 扩展为支持 `${表达式}`，与现有表达式语法一致，并保持向后兼容。

## 当前情况

- VOX 脚本中 speak/BGM 等文本支持 `${变量名}`，仅做变量名查找替换。
- 不支持 `${a + b}`、`${format_time(ts)}`、`${i + 1}` 等表达式。
- GUI 已有 ExprInput、变量/内置函数插入；engine/runner 已有完整表达式解析与求值（`parse_expr_from_str`、`eval_expr`、`eval_builtin`）。

## 目标（User Stories）

### P1：Engine 层支持 `${表达式}`

- 在 speak 与 BGM 等使用 `interpolate_text` 的文本中，将 `${...}` 内内容解析为**表达式**（与 DSL 中 set/condition/times 等使用的表达式语法一致）。
- 求值后转为字符串再拼接进文本。
- **向后兼容**：仅变量名的 `${name}` 仍合法（作为表达式中的标识符求值）。
- 解析/求值失败时：提供明确错误信息（建议含行号或上下文），不静默保留原样（或可配置为保留原样并打日志）。

### P2：错误与可观测性

- 表达式解析失败或求值失败时，错误信息可被 runner/GUI 展示（行号、片段、原因）。
- 符合宪章「可观测性」：关键路径有清晰错误提示。

### P3：文档与帮助

- README 与 GUI 帮助页中更新「文本插值」说明：支持 `${表达式}`，并给出示例（如 `${i}`、`${i + 1}`、`${format_time(ts)}`）。

## 约束

- 遵守 VoxStage 宪章：剧本优先与语法兼容、engine/runner/GUI 分层、可观测性、简单与资源受控。
- 不改变现有表达式语法；仅扩展「插值内容」从「变量名」到「任意合法表达式」。
- 若 `${...}` 内包含字符串字面量或括号，需正确识别匹配的 `}`（避免将 `"}"` 内的 `}` 当作结束）。

## 非目标（本迭代不做）

- GUI 端对 `${表达式}` 的专门可视化编辑（可后续迭代）。
- 新增内置函数或新语法。
