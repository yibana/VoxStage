# Contract: 文本插值语法与错误行为

## 语法契约

- **插值占位符**: `${` 与 匹配的 `}` 之间的内容为**表达式**，与 DSL 中 set/condition/times 等使用的表达式语法一致。
- **匹配规则**: 结束 `}` 的确定需考虑：
  - 双引号 `"..."` 与单引号 `'...'` 内的字符不参与 `}` 匹配；
  - 可选的 `{`/`}` 嵌套（若未来表达式支持块）时，按深度匹配。
- **兼容性**: 仅变量名的 `${name}` 为合法表达式的子集（标识符求值），行为与现有一致。

## 错误契约

| 情况 | 行为 |
|------|------|
| 提取 `${...}` 时未找到匹配的 `}` | 返回错误或保留至行尾的合理边界，并附带“未闭合的 ${”类描述。 |
| `parse_expr_from_str(inner)` 失败 | 返回错误，携带行号（或调用方传入的上下文）与 dsl 解析错误信息。 |
| `eval_expr(expr, vars)` 产生运行时错误（如除零、类型不符） | 返回错误，携带上下文与原因。 |
| 求值结果为非字符串（当前仅为 String/Num） | 按现有 `value_to_string` 规则转为字符串，不单独报错。 |

调用方（runner/GUI）可根据错误类型决定：打日志、展示给用户、或中止执行。
